🔍 调试空白页面问题

问题分析

页面显示空白通常由以下原因造成：

  • JavaScript 运行时错误
  • React 组件渲染错误
  • 路由配置问题
  • 认证状态异常
  • API 请求失败
  • CSS 样式问题

🚨 立即检查步骤

  1. 打开浏览器控制台 (F12)
    • 查看 Console 标签页是否有红色错误信息
    • 查看 Network 标签页确认 API 请求状态
  2. 检查认证状态
    • 在控制台执行: localStorage.getItem('auth-storage')
    • 确认是否有有效的认证信息
  3. 检查路由状态
    • 在控制台执行: window.location.pathname
    • 确认当前路径是否正确
  4. 检查组件渲染
    • 查看是否有 "🔥 BottomNavLayout is now rendering!" 的日志
    • 如果没有，说明组件没有渲染

🔧 快速修复方案

方案 1: 清除认证状态

在浏览器控制台执行：

localStorage.clear();
sessionStorage.clear();
location.reload();

方案 2: 直接访问登录页

访问: http://localhost:5173/auth/login

方案 3: 重启开发服务器

# 停止服务器 (Ctrl+C)
cd packages/frontend
npm run dev

方案 4: 临时回退到 ModernLayout

如果问题持续，可以临时回退到 ModernLayout 进行调试

📋 常见错误和解决方案

错误: Cannot read property 'xxx' of undefined

原因: 组件依赖的数据或 hook 返回 undefined

解决: 检查 useAuthStore, usePermission 等 hook 的返回值

错误: Module not found

原因: 导入路径错误或文件不存在

解决: 检查所有 import 语句的路径

错误: Hydration failed

原因: 服务端渲染和客户端渲染不匹配

解决: 清除缓存并重新启动

警告: 页面无限重定向

原因: 认证逻辑导致的重定向循环

解决: 检查 AuthLayout 和 ProtectedRoute 的逻辑
