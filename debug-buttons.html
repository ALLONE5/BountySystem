调试备份前端表格按钮问题

问题描述

5173（工作前端）显示所有操作按钮，但5174（备份前端）只显示"编辑"按钮

调试步骤

步骤 1: 打开浏览器开发者工具（F12）
步骤 2: 在控制台中运行以下代码来检查 TaskListTable 接收的 props： 
// 在 5174 端口的页面控制台中运行
// 找到 TaskListTable 组件实例
const tables = document.querySelectorAll('.task-table-wrapper');
console.log('找到的表格数量:', tables.length);

// 检查操作列
const actionCells = document.querySelectorAll('td[data-index="action"]');
console.log('操作列单元格数量:', actionCells.length);

// 检查按钮
const buttons = document.querySelectorAll('.action-buttons button');
console.log('找到的按钮数量:', buttons.length);
buttons.forEach((btn, i) => {
    console.log(`按钮 ${i}:`, btn.textContent, btn.className);
});
步骤 3: 添加临时调试日志到 TaskListTable.tsx

可能的原因

  • Props 没有正确传递到 TaskListTable
  • user 对象为 null 或 undefined
  • 任务状态不符合按钮显示条件
  • isPublishedTasksPage 标志未正确设置
  • 回调函数（onPublishTask, onEditTask 等）未定义
