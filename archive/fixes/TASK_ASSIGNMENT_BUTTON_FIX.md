# 任务指派按钮显示问题修复

## 更新日期
2026-02-03

## 问题描述
用户报告在"我的悬赏"页面的任务列表中看不到"指派"按钮。

## 根本原因分析

经过代码审查，发现以下问题：

### 1. 冗余列导致的布局问题
`PublishedTasksPage.tsx` 中存在两个显示承接者信息的列：
- **承接者列**（第 370 行）：显示详细的承接者信息（头像、用户名或组名）
- **承接人列**（第 400 行）：仅显示"已承接"或"未承接"文本

这个冗余列占用了表格空间，可能导致操作列的按钮显示不全。

### 2. 操作列宽度可能不足
操作列宽度设置为 250px，包含 4 个按钮（查看、指派、编辑、删除），在某些情况下可能显示不全。

## 已实施的修复

### 修复 1：移除冗余的"承接人"列
```typescript
// 删除了这个冗余列：
{
  title: '承接人',
  dataIndex: 'assigneeId',
  key: 'assigneeId',
  width: 100,
  render: (assigneeId: string | null) => assigneeId ? '已承接' : '未承接',
},
```

**理由**：
- "承接者"列已经清楚地显示了承接状态
- 未分配时显示"未分配"
- 已分配时显示用户头像和用户名
- 组任务时显示组名
- 信息更丰富，无需额外的"承接人"列

### 修复 2：增加操作列宽度
```typescript
{
  title: '操作',
  key: 'action',
  width: 280,  // 从 250px 增加到 280px
  fixed: 'right',
  // ...
}
```

**理由**：
- 确保所有按钮都能正常显示
- 避免按钮被挤压或隐藏

## 指派按钮显示逻辑

指派按钮通过 `canAssign()` 函数控制显示：

```typescript
const canAssign = (task: Task) => {
  // 只能指派未承接且状态合适的任务
  return !task.assigneeId && (
    task.status === TaskStatus.NOT_STARTED || 
    task.status === TaskStatus.AVAILABLE
  );
};
```

**显示条件**：
- ✅ 任务未被承接（`!task.assigneeId`）
- ✅ 任务状态为 `NOT_STARTED` 或 `AVAILABLE`

**不显示的情况**：
- ❌ 任务已有承接者
- ❌ 任务状态为 `IN_PROGRESS`、`COMPLETED` 等其他状态
- ❌ 任务状态为 `PENDING_ACCEPTANCE`（已发出邀请但未接受）

## 使用指南

### 如何指派任务

1. **进入"我的悬赏"页面**
   - 点击左侧菜单的"我的悬赏"

2. **找到要指派的任务**
   - 查看任务列表
   - 确认任务的"承接者"列显示"未分配"
   - 确认任务状态为"未开始"或"可承接"

3. **点击"指派"按钮**
   - 在操作列中找到"指派"按钮
   - 点击打开指派模态框

4. **搜索并选择用户**
   - 在搜索框中输入用户名或邮箱（至少 2 个字符）
   - 从下拉列表中选择目标用户
   - 注意：不能指派给自己

5. **确认指派**
   - 点击"确认指派"按钮
   - 系统会：
     - 将任务设置为私有
     - 发送邀请通知给被指派用户
     - 任务状态变为"待接受"

### 被指派用户的操作

1. **收到通知**
   - 被指派用户会收到任务邀请通知

2. **查看邀请**
   - 点击通知或进入"任务邀请"页面
   - 查看任务详情

3. **接受或拒绝**
   - **接受**：任务状态变为"进行中"，用户成为承接者
   - **拒绝**：任务恢复为"可承接"状态，可以重新指派

## 故障排除

### 问题：仍然看不到指派按钮

**可能原因 1：前端未重新编译**
```bash
# 重启前端服务
cd packages/frontend
npm run dev
```

**可能原因 2：任务状态不符合条件**
- 检查任务是否已有承接者
- 检查任务状态是否为 NOT_STARTED 或 AVAILABLE

**可能原因 3：浏览器缓存**
- 强制刷新页面（Ctrl+Shift+R）
- 清除浏览器缓存

**可能原因 4：表格横向滚动**
- 检查表格是否有横向滚动条
- 尝试向右滚动查看操作列

### 问题：指派后用户没有收到通知

**检查项**：
1. 后端通知服务是否正常运行
2. 被指派用户是否存在
3. 检查数据库 `notifications` 表

### 问题：指派失败

**常见错误**：
- "Cannot invite yourself" - 不能指派给自己
- "Task is already assigned" - 任务已被承接
- "Only the publisher can assign this task" - 只有发布者可以指派
- "Invited user not found" - 被邀请用户不存在

## 总结

通过移除冗余的"承接人"列和增加操作列宽度，我们解决了指派按钮可能不显示的问题。现在：

✅ 表格布局更清晰
✅ 操作列有足够空间显示所有按钮
✅ 指派按钮在符合条件的任务上正确显示
✅ 指派功能完整可用
