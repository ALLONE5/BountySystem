## 🎯 赏金算法说明

### 算法参数

1. **基础金额 (baseAmount)**
   - 所有任务的基础赏金
   - 单位：人民币（¥）

2. **紧急度权重 (urgencyWeight)**
   - 根据截止日期计算的紧急度系数（1-5）
   - 紧急度计算规则：
     - ≤ 1天：5（非常紧急）
     - ≤ 3天：4（紧急）
     - ≤ 7天：3（中等）
     - ≤ 14天：2（较低）
     - > 14天：1（最低）

3. **重要度权重 (importanceWeight)**
   - 任务优先级系数（1-5）
   - 由任务创建者设置

4. **工时权重 (durationWeight)**
   - 预估工时系数
   - 单位：小时

### 计算公式

```
赏金 = 基础金额 + (紧急度 × 紧急度权重) + (重要度 × 重要度权重) + (工时 × 工时权重)
```

### 示例计算

假设算法参数：
- 基础金额：100
- 紧急度权重：10
- 重要度权重：20
- 工时权重：5

**场景 1**: 低优先级，长期任务
- 紧急度：1，重要度：2，工时：5小时
- 赏金 = 100 + (1×10) + (2×20) + (5×5) = 175

**场景 2**: 中等优先级，中期任务
- 紧急度：3，重要度：3，工时：10小时
- 赏金 = 100 + (3×10) + (3×20) + (10×5) = 240

**场景 3**: 高优先级，紧急任务
- 紧急度：5，重要度：5，工时：20小时
- 赏金 = 100 + (5×10) + (5×20) + (20×5) = 350

## 📊 算法版本管理

### 版本隔离

- 每个任务存储其使用的算法版本
- 算法更改不影响已创建的任务
- 新任务使用当前生效的算法

### 生效规则

- 算法按 `effectiveFrom` 时间生效
- 系统自动选择最新的生效算法
- 可以提前创建未来生效的算法

### 已结算赏金保护

- 已结算的任务赏金不可更改
- `is_bounty_settled` 标志防止重新计算
- 确保支付完整性

## 📝 使用说明

### 创建新算法

1. 点击"创建新算法"按钮
2. 填写算法参数：
   - 版本号（例如：v2.0）
   - 基础金额（例如：100）
   - 各项权重（例如：10, 20, 5）
   - 计算公式（自动填充）
   - 生效时间（选择日期时间）
3. 点击"创建"保存

### 查看算法详情

1. 点击表格中的版本号
2. 查看完整的算法参数
3. 查看赏金计算示例
4. 了解算法的生效时间

### 算法生效

- 新算法在设定的生效时间后自动生效
- 新创建的任务将使用新算法计算赏金
- 已存在的任务继续使用原算法

## 🔄 与现有系统集成

### TaskService 集成

任务创建时自动调用赏金计算：
```typescript
const bountyCalculation = await bountyService.calculateBounty({
  estimatedHours,
  complexity,
  priority,
  plannedStartDate,
  plannedEndDate,
});
```

### 自动重新计算

当任务属性变更时自动重新计算赏金：
- 预估工时变更
- 复杂度变更
- 优先级变更
- 计划时间变更

**条件**: 仅当赏金未结算时重新计算
