问题诊断

问题根因：

在 TaskListContainer.tsx 中，props 传递顺序错误导致父组件传入的 action handlers 被内部的 handleCompleteTask 覆盖。

// 错误的顺序（修复前）：
<TaskListTable
  onCompleteTask={handleCompleteTask}  // 这个会被覆盖
  {...actionProps}  // 包含所有 action handlers
/>

// 正确的顺序（修复后）：
<TaskListTable
  {...actionProps}  // 先展开
  onCompleteTask={actionProps.onCompleteTask || handleCompleteTask}  // 优先使用传入的
/>

修复内容

✅ 修复文件： packages/frontend/src/components/TaskList/TaskListContainer.tsx

调整了 props 传递顺序，确保父组件传入的 action handlers 优先级更高。

✅ 清理调试代码：
  • 移除了 PublishedTasksPage.tsx 中的 console.log
  • 移除了 TaskListTable.tsx 中的调试日志

技术说明

Props 传递链： 
PublishedTasksPage
  ↓ (传递所有 action handlers)
TaskListPage
  ↓ (透传 props)
TaskListContainer
  ↓ (现在正确传递 actionProps)
TaskListTable
  ↓ (渲染按钮)
关键修复点：

使用 actionProps.onCompleteTask || handleCompleteTask 确保：

  • 如果父组件传入了 onCompleteTask，使用父组件的
  • 如果没有传入，使用 TaskListContainer 内部的默认实现
  • 这样既保持了向后兼容，又修复了按钮缺失问题
