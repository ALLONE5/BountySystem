## 为什么归档？

这些报告记录了项目的历史变更，但对日常开发参考价值有限。归档的目的是：

1. **保持文档结构清晰**: 减少主文档目录的文件数量
2. **保留历史记录**: 完整保存项目演进历史
3. **便于查找**: 需要时可以在归档中查找
4. **改善导航**: 让开发者更容易找到当前相关文档

## 如何使用归档

### 查找特定问题的修复记录
```bash

grep -r "关键词" docs/reports/archive/fixes/
```

### 了解某个功能的优化历史
```bash
# 查看优化报告
ls docs/reports/archive/optimizations/
```

### 学习重构模式
```bash
# 查看重构报告
ls docs/reports/archive/refactors/
```
