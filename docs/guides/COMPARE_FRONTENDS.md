# 前端对比指南

本指南说明如何同时运行两个前端版本进行对比。

---

## 📋 前提条件

- 后端服务已启动 (端口 3001)
- 数据库和 Redis 已运行

---

## 🚀 快速开始

### 方法一：使用对比脚本（推荐）

```bash
# 1. 安装备份前端的依赖（首次运行需要）
cd packages/frontend-bak
npm install
cd ../..

# 2. 启动两个前端进行对比
npm run dev:compare
```

### 方法二：手动启动

在不同的终端窗口中分别运行：

```bash
# 终端 1 - 当前前端
npm run dev:frontend

# 终端 2 - 备份前端
npm run dev:frontend-bak
```

---

## 🌐 访问地址

启动后，你可以在浏览器中打开：

- **当前前端**: http://localhost:5173
- **备份前端**: http://localhost:5174

两个前端都连接到同一个后端 API (http://localhost:3001)，使用相同的数据库。

## 🛠️ 配置说明

### 端口配置

- **当前前端**: `packages/frontend/vite.config.ts` - 端口 5173
- **备份前端**: `packages/frontend-bak/vite.config.ts` - 端口 5174

### API 代理配置

两个前端都配置了相同的 API 代理：

```typescript
proxy: {
  '/api': {
    target: 'http://localhost:3001',
    changeOrigin: true,
  },
}
```

## 💡 对比技巧

### 1. 并排窗口对比

使用 Windows 的窗口管理功能：
- 按 `Win + 左箭头` 将一个浏览器窗口固定到左侧
- 按 `Win + 右箭头` 将另一个浏览器窗口固定到右侧

### 2. 使用相同的测试账号

两个前端使用相同的后端，所以可以：
- 在一个前端登录
- 在另一个前端也登录相同账号
- 同步查看数据变化

### 3. 截图对比

使用浏览器扩展或工具截图保存：
- 相同页面的不同版本
- 相同操作的不同结果
- 错误状态的差异

## 🐛 常见问题

### 问题 1: 端口冲突

**症状**: 启动时提示端口已被占用

**解决方案**:
```bash
# 检查端口占用
netstat -ano | findstr :5173
netstat -ano | findstr :5174

# 如果需要，修改 vite.config.ts 中的端口号
```

### 问题 2: 依赖版本不一致

**症状**: 备份前端启动失败或功能异常

**解决方案**:
```bash
# 重新安装备份前端的依赖
cd packages/frontend-bak
rm -rf node_modules package-lock.json
npm install
```

### 问题 3: API 连接失败 (ERR_CONNECTION_REFUSED)

**症状**: 控制台显示 `GET http://localhost:3000/api/... net::ERR_CONNECTION_REFUSED`

**原因**: 备份前端配置的后端端口不正确

**解决方案**:
1. 确认后端运行在 3001 端口
2. 检查 `packages/frontend-bak/.env` 文件：
   ```
   VITE_API_URL=http://localhost:3001/api
   ```
3. 检查 `packages/frontend-bak/src/api/client.ts` 中的 baseURL
4. 重启备份前端服务

### 问题 4: 两个前端都无法获取数据

**症状**: 两个前端都无法获取数据

**解决方案**:
- 确认后端服务正在运行 (http://localhost:3001)
- 检查数据库连接
- 查看后端日志

### 问题 5: 备份前端登录成功后不跳转

**症状**: 登录成功显示消息，但停留在登录页面不跳转到 dashboard

**原因**: React 状态更新竞态条件导致路由守卫检查时认证状态未更新

**解决方案**: 
- 此问题已在 2026-03-12 修复
- 如果仍遇到问题，请清除浏览器缓存和 localStorage
- 详细信息见 [备份前端登录修复报告](../reports/FRONTEND_BAK_LOGIN_FIX.md)
