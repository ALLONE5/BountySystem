## 任务管理

### 任务状态流转
```
not_started → available → pending_acceptance → in_progress → completed
                                              ↓
                                          abandoned
```

### 任务可见性
- **PUBLIC**: 所有用户可见
- **POSITION_ONLY**: 仅特定岗位用户可见
- **PRIVATE**: 仅发布者和受邀用户可见

### 任务深度限制
- **最大深度**: 1 层（depth 0-1）
- **depth 0**: 顶级任务（母任务）
- **depth 1**: 一级子任务
- **不支持**: 孙任务（子任务的子任务）

### 任务分配工作流

#### 方式1: 公开任务
```
发布者创建任务 → 发布为公开 → 用户浏览并申请
→ 发布者审核 → 批准后任务状态变为 pending_acceptance
→ 用户接受 → 任务状态变为 in_progress
```

#### 方式2: 邀请任务
```
发布者创建任务 → 邀请特定用户 → 用户收到通知
→ 用户接受邀请 → 任务状态变为 in_progress
```

### 任务完成和放弃

#### 完成任务
- 执行者标记任务为完成
- 系统自动分配赏金
- 更新用户余额
- 更新排名数据（异步）
- 发送通知给发布者

#### 放弃任务
- **子任务放弃**: 清空执行者，任务状态变为 available，可重新承接
- **母任务放弃**: 
  - 保留已完成子任务的执行者和状态
  - 清空未完成子任务的执行者
  - 母任务状态变为 available

### 任务进度自动更新
- 子任务完成时，自动更新母任务进度
- 进度计算：已完成子任务数 / 总子任务数 × 100%
- 实时同步到前端（WebSocket）

### 任务列表项目分组
- 支持按项目组分组显示任务
- 可折叠/展开项目组
- 显示每个项目组的任务数量
- 支持在看板、甘特图、日历视图中使用

### 浏览任务可见性逻辑

**可见性规则**:
1. **PUBLIC 任务**: 所有用户可见
2. **POSITION_ONLY 任务**: 仅该岗位的用户可见
3. **PRIVATE 任务**: 仅发布者和受邀用户可见

**过滤条件**:
- 必须是已发布的任务 (`is_published = true`)
- 必须是可用状态 (`status = 'available'`)
- 不显示已被承接的任务
- 不显示用户自己发布的任务

### 我的任务页面逻辑

**显示内容**:
- 当前用户承接的所有任务
- 包括母任务和子任务
- 使用递归 CTE 查询获取完整任务树

**SQL 查询逻辑**:
```sql
WITH RECURSIVE task_tree AS (
  -- 基础查询：用户直接承接的任务
  SELECT * FROM tasks WHERE assignee_id = $1
  UNION ALL
  -- 递归查询：这些任务的所有子任务
  SELECT t.* FROM tasks t
  INNER JOIN task_tree tt ON t.parent_id = tt.id
)
SELECT * FROM task_tree;
```

### 已发布任务子任务显示逻辑

**显示规则**:
- 仅显示顶层任务（depth = 0）
- 子任务通过展开按钮查看
- 子任务数量显示在徽章中
- 支持递归加载子任务树

## 子任务系统

### 子任务创建要求

**前置条件**:
- 母任务必须已被承接（有 assignee_id）
- 母任务承接者才能创建子任务
- 子任务深度不能超过 1

**继承属性**:
- 项目组（project_group_id）
- 可见性（visibility）
- 组群（group_id，如果有）

**独立属性**:
- 标题、描述
- 赏金金额
- 计划开始/结束时间
- 优先级、复杂度
- 承接者（可以不同于母任务承接者）

### 子任务发布到公开

**功能**:
- 子任务创建后默认为草稿状态
- 母任务承接者可以将子任务发布为公开任务
- 发布后其他用户可以浏览和申请承接

**限制**:
- 只有母任务承接者可以发布子任务
- 子任务必须满足基本验证（标题、描述等）

### 子任务删除

**删除规则**:
- 只有未承接的子任务可以删除
- 已承接或已完成的子任务不能删除
- 删除子任务会更新母任务的子任务计数

### 子任务视图实现

**视图类型**:
1. **列表视图**: 平铺显示所有子任务
2. **树形视图**: 层级结构显示
3. **看板视图**: 按状态分列显示
4. **甘特图**: 时间轴显示

### 子任务详情视图

**显示方式**: 全屏抽屉（Drawer）
**内容**:
- 子任务完整信息
- 母任务信息（面包屑导航）
- 操作按钮（编辑、删除、完成等）
- 评论和附件（如果有）

### 子任务表单标准化

**统一字段**:
- 标题（必填，3-200 字符）
- 描述（必填）
- 赏金金额（必填，> 0）
- 计划开始时间（可选）
- 计划结束时间（可选）
- 优先级（低/中/高）
- 复杂度（简单/中等/复杂）

### 子任务深度限制和 UI 改进

**深度限制**:
- 最大深度：1 层
- 不允许创建孙任务
- UI 上禁用子任务的"创建子任务"按钮

**UI 改进**:
- 显示任务层级（面包屑）
- 子任务计数徽章
- 折叠/展开子任务列表
- 层级缩进显示

### 子任务承接人修复

**问题**: 子任务承接人显示不正确
**原因**: SQL 查询中 JOIN 条件错误
**解决**: 修复 LEFT JOIN 条件，正确关联 assignee 用户信息

### 子任务发布工作流状态

**状态**: ✅ 已完成
**功能**:
- 子任务创建后为草稿状态
- 母任务承接者可以发布子任务
- 发布后子任务进入可用状态
- 其他用户可以浏览和申请

## 岗位管理

### 岗位管理编辑功能

**功能**:
- 编辑岗位名称和描述
- 修改岗位管理员
- 管理岗位成员
- 设置岗位权限

**权限**:
- 超级管理员可以编辑所有岗位

### 岗位申请审核 UI 改进

**改进内容**:
- 显示申请者详细信息
- 显示申请时间
- 批准/拒绝按钮

**审核流程**:
```

→ 管理员审核 → 批准/拒绝
→ 用户收到审核结果通知
```

**问题**: 移除岗位成员时出错
**原因**: 外键约束和权限检查问题
**解决**: 
- 修复外键约束处理
- 添加权限检查
- 更新缓存失效逻辑

**场景**: 岗位管理员离职，需要替换
**流程**:
```
选择新管理员 → 验证权限 → 更新岗位管理员
→ 通知新旧管理员 → 更新相关权限
```

## 组群管理

### 组群解散功能

**解散规则**:
- 组群名称从"删除"改为"解散"
- 如果组群有已完成的任务，不能解散
- 只能解散没有已完成任务的组群

**验证逻辑**:
```typescript
// 检查是否有已完成的任务
const completedTasks = await checkCompletedTasks(groupId);
if (completedTasks > 0) {
  throw new ValidationError('组群中有已完成的任务，不能解散');
}
```

### 组群任务顶层过滤

**功能**: 在组群任务列表中只显示顶层任务
**实现**: 添加 `parent_id IS NULL` 过滤条件
**优势**: 
- 减少列表混乱
- 子任务通过展开查看
- 提高性能

### 组群任务承接人逻辑更新

**更新内容**:
- 组群任务可以由组群成员承接
- 非组群成员不能承接组群任务
- 承接人必须在组群中

## 通知系统

### 通知系统审核逻辑

**通知类型**:
- `review_required`: 岗位申请待审核
- `position_approved`: 岗位申请已批准
- `position_rejected`: 岗位申请已拒绝
- `task_assignment_invitation`: 任务分配邀请
- `task_completed`: 任务已完成
- `task_abandoned`: 任务已放弃

**审核流程**:
```
用户提交申请 → 创建 review_required 通知
→ 推送给岗位管理员（WebSocket）
→ 管理员审核 → 创建审核结果通知
→ 推送给申请者
```

**实时推送**:
- 使用 WebSocket 实时推送通知
- 用户在线时立即收到
- 离线时下次登录拉取

## 管理员功能

### 管理员审批徽章功能

**功能**: 在管理员侧边栏"审核操作"菜单项上显示待审核数量徽章

**实现**:
- 查询 `status = 'pending'` 的岗位申请数量
- 显示红色徽章
- 实时更新（WebSocket + 60秒轮询）

### 管理员审批徽章 WebSocket 同步

**优化**: 从轮询改为 WebSocket 实时同步

**实现**:
```typescript
// 监听通知事件
socket.on('application-submitted', () => {
  fetchPendingCount();
});

socket.on('application-reviewed', () => {
  fetchPendingCount();
});
```

**效果**: 更新延迟从 60 秒降低到 <1 秒
