# 管理功能菜单徽章同步

## 修改日期
2026年2月10日

## 问题描述
当"审核操作"子菜单项显示红点（待审核数量徽章）时，父级"管理功能"菜单项没有显示相应的徽章，导致用户在菜单折叠时无法看到有待处理的审核任务。

## 解决方案
在"管理功能"父级菜单项上也显示相同的待审核数量徽章，与"审核操作"子菜单项保持同步。

## 实现细节

### 修改文件
- `packages/frontend/src/layouts/MainLayout.tsx`

### 代码变更

**修改前**:
```typescript
{
  key: 'admin',
  label: '管理功能',
  children: adminChildren.filter((item) => {
    // ... filter logic
  }),
}
```

**修改后**:
```typescript
{
  key: 'admin',
  label: (
    <Space>
      管理功能
      {pendingApplicationCount > 0 && (
        <Badge count={pendingApplicationCount} size="small" />
      )}
    </Space>
  ),
  children: adminChildren.filter((item) => {
    // ... filter logic
  }),
}
```

## 徽章显示逻辑

### 数据来源
- `pendingApplicationCount`: 从 `adminApi.getApplications()` 获取状态为 'pending' 的申请数量

### 更新机制
1. **初始加载**: 组件挂载时加载待审核数量
2. **事件监听**:
   - `application-reviewed`: 审核操作完成后触发
   - `application-submitted`: 新申请提交后触发（通过 WebSocket）
3. **定时刷新**: 每 60 秒自动刷新一次（防止 WebSocket 事件丢失）

### 显示位置
- **父级菜单**: "管理功能" - 显示待审核总数
- **子级菜单**: "审核操作" - 显示待审核总数

两个位置显示相同的数量，确保用户在任何状态下都能看到待处理的审核任务。

  ├─ 审核操作 🔴 3    ← 子级菜单显示徽章

### 优势
1. **更明显**: 即使菜单折叠，管理员也能立即看到有待处理的审核任务
2. **一致性**: 父子菜单徽章数量保持同步，避免混淆
3. **及时性**: 通过多种机制（事件监听、定时刷新）确保徽章数量实时更新

## 权限控制
徽章仅对有管理权限的用户显示：
- `UserRole.SUPER_ADMIN`: 超级管理员
- `UserRole.POSITION_ADMIN`: 岗位管理员

通过 `canAccessAdminPanel()` 方法进行权限检查。

## 测试建议

### 功能测试
1. 以管理员身份登录
2. 提交一个岗位变更申请（使用普通用户账号）
3. 验证"管理功能"和"审核操作"都显示徽章
4. 审核申请后，验证两个徽章都消失或数量减少
5. 测试菜单折叠/展开状态下徽章的显示

### 边界测试
1. 待审核数量为 0 时，徽章不显示
2. 待审核数量为 1 时，徽章显示 "1"
3. 待审核数量超过 99 时，徽章显示 "99+"
4. 多个申请同时提交时，徽章数量正确累加

### 性能测试
1. 验证定时刷新不会造成性能问题
2. 验证 WebSocket 事件正确触发更新
3. 验证组件卸载时清理定时器和事件监听器

## 相关功能
- 审核操作页面: `/admin/approval`
- 岗位变更申请: `ProfilePage` 中的"申请岗位变更"功能
- 通知系统: 审核结果通过通知系统推送给用户

## 注意事项
1. 徽章数量基于 `status === 'pending'` 的申请过滤
2. 仅统计岗位变更申请，不包括其他类型的审核
3. 徽章使用 Ant Design 的 `Badge` 组件，样式为 `size="small"`
