# 任务详情子任务多视图功能实现

## 概述
为任务详情页面的子任务界面添加了多种视图支持，包括列表、甘特图、看板和日历视图，提升用户体验和任务管理效率。

## 功能描述

### 新增视图类型
在任务详情的"子任务"标签页中，用户现在可以通过以下四种视图查看子任务：

1. **列表视图** - 传统的列表展示方式，显示子任务的详细信息
2. **甘特图视图** - 时间轴展示，直观显示子任务的时间安排和依赖关系
3. **看板视图** - 按状态分组的卡片展示，适合敏捷开发流程
4. **日历视图** - 日历形式展示，方便查看任务的时间分布

### 视图切换
- 用户可以通过标签页快速切换不同视图
- 每个视图都保持相同的数据源（子任务列表）
- 视图切换不会丢失当前的操作状态

## 实现细节

### 1. 组件导入
```typescript
import { TaskViews } from './TaskViews';
```

### 2. 子任务渲染函数重构
将原来的简单列表渲染改为使用 `TaskViews` 组件：

```typescript
const renderSubtasks = () => {
  // 创建子任务列表视图
  const subtaskListView = (
    <List
      dataSource={subtasks}
      locale={{ emptyText: '暂无子任务' }}
      renderItem={(sub) => (
        // ... 列表项渲染逻辑
      )}
    />
  );

  return (
    <Card bordered={false} bodyStyle={{ padding: 0 }}>
      <TaskViews
        tasks={subtasks}
        loading={false}
        listView={subtaskListView}
        extra={
          // 创建子任务按钮
        }
      />
    </Card>
  );
};
```

### 3. TaskViews 组件配置
- **tasks**: 传入子任务数组
- **loading**: 加载状态（设为 false，因为子任务已在父组件加载）
- **listView**: 自定义的列表视图组件
- **extra**: 额外的操作按钮（创建子任务按钮）

### 4. 列表视图保留功能
列表视图保留了所有原有功能：
- 查看子任务详情（Popover）
- 发布子任务
- 删除子任务
- 显示任务状态、进度、优先级
- 显示承接者信息

### 各视图特点

- 显示子任务名称、状态、优先级

```
└─ Calendar View (CalendarPage)
```

## 使用场景

### 场景 1: 项目时间规划
用户可以切换到甘特图视图，查看所有子任务的时间安排，识别时间冲突和资源分配问题。

### 场景 2: 敏捷开发管理
团队使用看板视图，将子任务按状态分组，快速了解任务进展，识别瓶颈。

### 场景 3: 日程安排
用户切换到日历视图，查看特定日期的任务分布，合理安排工作计划。

### 场景 4: 详细信息查看
用户在列表视图中查看子任务的详细信息，进行编辑、发布等操作。

## 注意事项

### 1. 性能考虑
- 子任务数据在父组件中已加载，避免重复请求
- 视图切换不会重新加载数据
- 大量子任务时，甘特图和日历视图可能需要优化

### 2. 功能限制
- 只有顶层任务（depth 0）才显示子任务标签页
- 子任务（depth 1）不能再创建子任务
- 创建子任务按钮只在母任务被承接后才可用

### 3. 数据一致性
- 所有视图共享同一数据源
- 在任何视图中的操作都会同步更新
- 刷新数据时所有视图都会更新
