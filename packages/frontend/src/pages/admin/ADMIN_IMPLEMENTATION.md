# 管理员界面实现总结

## 概述

本文档总结了赏金猎人平台管理员界面的实现，包括用户管理、任务管理和岗位申请审核三个核心功能模块。

> 角色可见性更新：岗位管理员的侧边栏仅显示“用户管理 / 任务管理 / 审核操作”，头像管理、岗位管理、赏金算法仅对超级管理员可见。

## 实现的功能

### 1. 用户管理界面 (UserManagementPage)

**文件位置**: `packages/frontend/src/pages/admin/UserManagementPage.tsx`

**功能特性**:
- 显示用户列表（根据管理员权限自动过滤）
  - 超级管理员：查看所有用户
  - 职位管理员：仅查看拥有其管理岗位的用户
- 用户详情查看
  - 基本信息：用户名、邮箱、角色
  - 岗位信息：显示用户拥有的所有岗位
  - 系统信息：用户ID、创建时间、最后登录时间
- 用户信息编辑
  - 编辑用户名和邮箱
  - 超级管理员可以修改用户角色
- 用户删除（仅超级管理员）

**验证需求**: 15.1, 15.2, 15.3, 15.4, 15.5

### 2. 任务管理界面 (TaskManagementPage)

**文件位置**: `packages/frontend/src/pages/admin/TaskManagementPage.tsx`

**功能特性**:
- 显示任务列表（根据管理员权限自动过滤）
  - 超级管理员：查看所有任务
  - 职位管理员：仅查看与其管理岗位相关的任务
- 任务搜索和筛选
  - 按任务名称或描述搜索
  - 按任务状态筛选
- 任务详情查看
  - 基本信息：名称、描述、标签
  - 任务属性：复杂度、优先级、预估工时
  - 时间信息：计划开始/结束时间
  - 状态信息：任务状态、进度、赏金金额
- 任务编辑
  - 修改任务基本信息和属性
  - 调整时间安排
  - 更改可见性设置
- 任务删除

**验证需求**: 16.1, 16.2, 16.3, 16.4, 16.5

### 3. 岗位申请审核界面 (ApplicationReviewPage)

**文件位置**: `packages/frontend/src/pages/admin/ApplicationReviewPage.tsx`

**功能特性**:
- 显示待审核岗位申请列表
  - 超级管理员：查看所有待审核申请
  - 职位管理员：仅查看其管理岗位的申请
- 申请详情查看
  - 申请人信息：用户名、邮箱、用户ID
  - 岗位信息：岗位名称、描述、岗位ID
  - 申请信息：申请理由、申请时间、审核状态
- 审核操作
  - 批准申请：可选填写审核意见
  - 拒绝申请：必须填写拒绝理由
  - 审核后显示审核时间和审核意见
  - 有待审核申请时，管理员通知中心会收到 “审核提醒” 类型通知（REVIEW_REQUIRED），提醒超级管理员及对应岗位管理员处理。

**验证需求**: 17.1, 17.2, 17.3, 17.4, 17.5

## API 客户端

**文件位置**: `packages/frontend/src/api/admin.ts`

实现了与后端管理员API的完整对接：

### 用户管理 API
- `getUsers()`: 获取用户列表
- `getUserDetails(userId)`: 获取用户详情
- `updateUser(userId, data)`: 更新用户信息
- `deleteUser(userId)`: 删除用户

### 任务管理 API
- `getTasks()`: 获取任务列表
- `getTaskDetails(taskId)`: 获取任务详情
- `updateTask(taskId, data)`: 更新任务信息
- `deleteTask(taskId)`: 删除任务

### 岗位申请审核 API
- `getApplications()`: 获取待审核申请列表
- `getApplicationDetails(applicationId)`: 获取申请详情
- `reviewApplication(applicationId, data)`: 审核申请（批准/拒绝）

## 路由配置

**文件位置**: `packages/frontend/src/router/index.tsx`

添加了三个管理员路由：
- `/admin/users` - 用户管理界面
- `/admin/tasks` - 任务管理界面
- `/admin/approval` - 岗位申请审核界面

## 权限控制

所有管理员界面都通过以下机制进行权限控制：

1. **前端权限检查**: 使用 `usePermission` hook 检查用户角色
2. **后端权限验证**: 所有API请求都在后端进行权限验证
3. **数据过滤**: 根据管理员类型自动过滤可见数据
   - 超级管理员：访问所有数据
  - 职位管理员：仅访问其管理岗位相关的数据；菜单仅显示用户管理、任务管理、审核操作

### 响应式布局
- 使用 Grid 布局实现左右分栏
- 左侧列表，右侧详情
- 移动端自动调整为单列布局

### 交互体验
- 列表项点击高亮显示
- 编辑模式切换
- 确认对话框防止误操作
- 加载状态提示
- 错误信息显示

### 视觉设计
- 状态标签使用不同颜色区分
- 进度条可视化显示任务进度
- 表单验证和提示
- 响应式按钮状态

## 技术栈

- **React**: 组件化开发
- **TypeScript**: 类型安全
- **React Router**: 路由管理
- **Zustand**: 状态管理（认证状态）
- **Axios**: HTTP 请求
- **Tailwind CSS**: 样式设计
