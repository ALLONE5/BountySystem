# 需求文档

## 简介

赏金猎人平台是一个任务发布与承接系统，支持多层级任务拆解、用户岗位管理、团队协作、自动赏金计算和多种任务可视化方式。该平台旨在通过悬赏机制激励用户完成任务，并提供完善的权限管理和任务调度功能。

## 术语表

- **系统 (System)**: 赏金猎人平台应用程序
- **用户 (User)**: 使用平台的个人，可以是普通用户、职位管理员或超级管理员
- **任务 (Task)**: 可以被发布和承接的工作单元
- **母任务 (Parent Task)**: 包含子任务的上级任务
- **子任务 (Subtask)**: 从母任务拆解出的下级任务
- **可执行任务 (Executable Task)**: 最底层的、可以被实际执行的任务
- **岗位 (Position)**: 用户的技能身份标识
- **职位管理员 (Position Administrator)**: 管理特定岗位的用户
- **超级管理员 (Super Administrator)**: 拥有系统最高权限的用户
- **任务组群 (Task Group)**: 多个用户组成的协作团队
- **赏金 (Bounty)**: 完成任务后获得的奖励金额
- **辅助用户 (Assistant User)**: 协助完成任务的用户
- **任务依赖 (Task Dependency)**: 任务之间的先后顺序关系
- **可见性 (Visibility)**: 任务对不同用户的可见范围

## 需求

### 需求 1：任务层级结构管理

**用户故事：** 作为任务发布者，我希望能够创建多层级的任务结构，以便将复杂项目拆解为可管理的子任务。

#### 验收标准

1. WHEN 用户创建任务 THEN 系统应允许该任务作为独立任务或作为现有任务的子任务
2. WHEN 任务被创建为子任务 THEN 系统应验证层级深度不超过三层
3. WHEN 用户尝试在第三层任务下创建子任务 THEN 系统应拒绝该操作并提示最大层级限制
4. WHEN 母任务包含子任务 THEN 系统应自动聚合并显示所有子任务的统计信息
5. WHERE 任务为最底层任务 THEN 系统应标记该任务为可执行任务

### 需求 2：任务属性管理

**用户故事：** 作为任务发布者，我希望为任务设置详细的属性信息，以便准确描述任务需求和约束条件。

#### 验收标准

1. WHEN 用户创建可执行任务 THEN 系统应要求输入任务名称和描述
2. WHEN 用户设置任务属性 THEN 系统应允许设置自定义标签、计划开始时间、计划结束时间、预估工时、复杂度、优先级、所属岗位和可见性
3. WHEN 任务被创建 THEN 系统应自动记录创建时间并根据任务属性计算悬赏金额
4. WHEN 任务状态变更为进行中 THEN 系统应记录实际开始时间
5. WHEN 任务状态变更为已完成 THEN 系统应记录实际结束时间
6. WHEN 母任务的子任务属性更新 THEN 系统应重新计算并更新母任务的聚合属性

### 需求 3：任务依赖关系管理

**用户故事：** 作为任务发布者，我希望设置任务之间的依赖关系，以便控制任务的执行顺序。

#### 验收标准

1. WHEN 用户为任务添加依赖 THEN 系统应允许指定一个或多个前置任务
2. WHEN 任务存在未完成的前置任务 THEN 系统应阻止该任务被承接
3. WHEN 前置任务完成 THEN 系统应自动解除依赖并通知相关用户
4. WHEN 用户尝试创建循环依赖 THEN 系统应拒绝该操作并提示错误
5. WHEN 任务依赖关系变更 THEN 系统应触发自动调度重新评估任务可用性

### 需求 4：用户岗位管理

**用户故事：** 作为用户，我希望申请和管理自己的岗位身份，以便承接符合我技能的任务。

#### 验收标准

1. WHEN 用户申请岗位 THEN 系统应将申请提交给对应的职位管理员审核
2. WHEN 用户拥有的岗位数量少于三个 THEN 系统应允许用户申请新岗位
3. WHEN 用户已拥有三个岗位 THEN 系统应拒绝新的岗位申请
4. WHEN 职位管理员审核岗位申请 THEN 系统应允许通过或拒绝该申请
5. WHEN 岗位申请被通过 THEN 系统应将该岗位添加到用户的岗位列表并通知用户
6. WHEN 岗位申请被拒绝 THEN 系统应通知用户并说明拒绝原因

### 需求 5：任务承接权限控制

**用户故事：** 作为普通用户，我希望只能看到和承接符合我岗位要求的任务，以确保任务分配的合理性。

#### 验收标准

1. WHEN 用户浏览任务列表 THEN 系统应只显示对该用户可见的任务
2. WHEN 任务设置了岗位要求 THEN 系统应只允许拥有该岗位的用户承接
3. WHEN 任务未设置岗位要求 THEN 系统应允许任何用户承接
4. WHEN 用户尝试承接不符合岗位要求的任务 THEN 系统应拒绝并提示岗位不匹配
5. WHEN 用户承接任务 THEN 系统应更新任务状态并通知任务发布者

### 需求 6：用户角色权限管理

**用户故事：** 作为系统管理员，我希望根据用户角色分配不同的权限，以确保系统的安全性和可管理性。

#### 验收标准

1. WHEN 普通用户登录 THEN 系统应允许访问个人界面、发布任务管理、承接任务管理、赏金任务界面和排名界面
2. WHEN 职位管理员登录 THEN 系统应额外允许访问用户管理界面、任务管理界面和审核操作界面
3. WHEN 超级管理员登录 THEN 系统应允许访问所有界面并拥有所有用户和任务的管理权限
4. WHEN 职位管理员查看用户信息 THEN 系统应只显示拥有其管理岗位的用户
5. WHEN 职位管理员编辑任务 THEN 系统应只允许编辑与其管理岗位相关的任务
6. WHEN 超级管理员执行操作 THEN 系统应允许查看、编辑和删除任何用户或任务

### 需求 7：任务组群协作

**用户故事：** 作为用户，我希望能够与其他用户组成团队共同承接任务，以便协作完成复杂项目。

#### 验收标准

1. WHEN 用户创建任务组群 THEN 系统应允许邀请其他用户加入
2. WHEN 任务组群承接任务 THEN 系统应使该任务对所有组群成员可见
3. WHEN 组群成员查看任务 THEN 系统应显示所有组群成员和任务详情
4. WHEN 用户加入或离开组群 THEN 系统应更新组群成员列表并通知所有成员
5. WHEN 组群任务完成 THEN 系统应按预设规则分配赏金给所有成员

### 需求 8：用户个人信息管理

**用户故事：** 作为用户，我希望能够管理自己的个人信息，以便保持账户信息的准确性。

#### 验收标准

1. WHEN 用户访问头像选择界面 THEN 系统应显示用户当前排名可解锁的所有头像
2. WHEN 用户上月排名确定 THEN 系统应根据排名等级解锁对应的头像库
3. WHEN 用户选择头像 THEN 系统应验证该头像是否在用户可选范围内
4. WHEN 用户尝试选择未解锁头像 THEN 系统应拒绝并提示所需排名要求
5. WHEN 用户修改邮箱 THEN 系统应验证邮箱格式并发送验证邮件
6. WHEN 用户修改密码 THEN 系统应要求输入当前密码并验证新密码强度
7. WHEN 用户更新个人信息 THEN 系统应保存更改并显示成功提示
8. WHEN 用户邮箱验证成功 THEN 系统应更新邮箱地址并通知用户

### 需求 9：任务发布管理界面

**用户故事：** 作为任务发布者，我希望有一个专门的界面管理我发布的所有任务，以便跟踪任务进展。

#### 验收标准

1. WHEN 用户访问发布任务管理界面 THEN 系统应显示该用户发布的所有任务列表
2. WHEN 用户选择任务 THEN 系统应显示任务详情和当前状态
3. WHEN 用户编辑已发布任务 THEN 系统应允许修改任务属性并保存更改
4. WHEN 任务已被承接 THEN 系统应限制某些属性的修改
5. WHEN 用户删除未承接的任务 THEN 系统应移除该任务并通知相关用户

### 需求 10：任务承接管理界面

**用户故事：** 作为任务承接者，我希望有一个专门的界面管理我承接的所有任务，以便组织我的工作。

#### 验收标准

1. WHEN 用户访问承接任务管理界面 THEN 系统应显示该用户承接的所有任务列表
2. WHEN 用户查看任务组群 THEN 系统应显示所有加入的组群列表
3. WHEN 用户点击组群 THEN 系统应显示该组群的所有任务和成员信息
4. WHEN 用户更新任务状态 THEN 系统应保存状态变更并通知任务发布者
5. WHEN 用户完成任务 THEN 系统应触发赏金结算流程

### 需求 11：任务操作管理

**用户故事：** 作为任务承接者，我希望能够对已承接的任务进行放弃、转让等操作，以便灵活管理工作负载。

#### 验收标准

1. WHEN 用户选择放弃任务 THEN 系统应将任务状态恢复为未承接并通知任务发布者
2. WHEN 用户选择转让任务 THEN 系统应允许指定接收用户并验证其岗位资格
3. WHEN 任务转让完成 THEN 系统应更新任务承接者并通知双方用户
4. WHEN 用户添加辅助用户 THEN 系统应允许选择按比例分配或固定数额分配赏金方式
5. WHEN 用户选择按比例分配 THEN 系统应要求输入辅助用户获得的赏金百分比
6. WHEN 用户选择固定数额分配 THEN 系统应要求输入辅助用户获得的固定赏金金额
7. WHEN 固定数额超过任务总赏金 THEN 系统应拒绝并提示金额不能超过任务赏金
8. WHEN 辅助用户被添加 THEN 系统应通知辅助用户并显示其将获得的赏金信息
9. WHEN 任务完成且有辅助用户 THEN 系统应按预设的分配方式计算并分配赏金给所有参与者

### 需求 12：个人界面概览

**用户故事：** 作为用户，我希望在个人界面看到所有任务的概览，以便快速了解工作状态。

#### 验收标准

1. WHEN 用户访问个人界面 THEN 系统应显示发布任务和承接任务的统计摘要
2. WHEN 用户点击任务摘要 THEN 系统应跳转到对应的任务管理界面
3. WHEN 用户选择生成报告 THEN 系统应允许选择日总结、周总结、月总结或总总结
4. WHEN 用户生成报告 THEN 系统应汇总相应时间段的任务数据并生成可下载的报告
5. WHEN 报告生成完成 THEN 系统应提供下载链接或在线预览功能

### 需求 13：赏金任务浏览界面

**用户故事：** 作为用户，我希望能够浏览所有可承接的赏金任务，以便找到适合我的工作机会。

#### 验收标准

1. WHEN 用户访问赏金任务界面 THEN 系统应显示所有对该用户可见的未承接任务
2. WHEN 用户输入关键字搜索 THEN 系统应过滤并显示匹配任务名称或描述的任务
3. WHEN 用户选择排序方式 THEN 系统应按赏金金额、截止日期、优先级或创建时间排序任务
4. WHEN 用户选择分组方式 THEN 系统应按岗位、标签或复杂度分组显示任务
5. WHEN 用户点击任务 THEN 系统应显示任务详情并提供承接选项

### 需求 14：排名界面

**用户故事：** 作为用户，我希望查看赏金排名，以便了解自己和其他用户的表现。

#### 验收标准

1. WHEN 用户访问排名界面 THEN 系统应默认显示本月赏金排名
2. WHEN 用户选择本季度排名 THEN 系统应显示当前季度的累计赏金排名
3. WHEN 用户选择总累积排名 THEN 系统应显示所有时间的累计赏金排名
4. WHEN 排名显示 THEN 系统应包含用户名、头像、赏金总额和排名位置
5. WHEN 用户在排名列表中 THEN 系统应高亮显示当前用户的排名
6. WHEN 每月结束 THEN 系统应根据上月排名更新所有用户的头像解锁权限

### 需求 15：管理员用户管理界面

**用户故事：** 作为管理员，我希望有专门的界面管理用户信息，以便维护系统用户数据。

#### 验收标准

1. WHEN 职位管理员访问用户管理界面 THEN 系统应显示拥有其管理岗位的所有用户
2. WHEN 超级管理员访问用户管理界面 THEN 系统应显示所有用户
3. WHEN 管理员查看用户详情 THEN 系统应显示用户的所有信息和任务历史
4. WHEN 管理员编辑用户信息 THEN 系统应允许修改用户属性并保存更改
5. WHEN 超级管理员删除用户 THEN 系统应移除用户账户并处理其相关任务

### 需求 16：管理员任务管理界面

**用户故事：** 作为管理员，我希望有专门的界面管理任务信息，以便监督和调整任务分配。

#### 验收标准

1. WHEN 职位管理员访问任务管理界面 THEN 系统应显示与其管理岗位相关的所有任务
2. WHEN 超级管理员访问任务管理界面 THEN 系统应显示所有任务
3. WHEN 管理员查看任务详情 THEN 系统应显示任务的所有属性和执行历史
4. WHEN 管理员编辑任务 THEN 系统应允许修改任务属性并保存更改
5. WHEN 管理员删除任务 THEN 系统应移除任务并通知相关用户

### 需求 17：审核操作界面

**用户故事：** 作为职位管理员，我希望有专门的界面处理岗位申请审核，以便高效管理岗位分配。

#### 验收标准

1. WHEN 职位管理员访问审核操作界面 THEN 系统应显示待审核的岗位申请列表
2. WHEN 管理员查看申请详情 THEN 系统应显示申请用户的信息和申请理由
3. WHEN 管理员通过申请 THEN 系统应授予用户岗位并发送通知
4. WHEN 管理员拒绝申请 THEN 系统应要求输入拒绝理由并通知用户
5. WHEN 审核操作完成 THEN 系统应从待审核列表中移除该申请

### 需求 18：任务点评和额外奖励

**用户故事：** 作为任务发布者或管理员，我希望能够对完成的任务进行点评并发放额外奖金，以便激励优秀表现。

#### 验收标准

1. WHEN 任务完成 THEN 系统应允许任务发布者和管理员添加点评
2. WHEN 用户添加点评 THEN 系统应允许输入文字评价和评分
3. WHEN 用户发放额外奖金 THEN 系统应允许输入奖金金额并验证可用余额
4. WHEN 额外奖金发放 THEN 系统应将金额添加到任务承接者的账户并记录交易
5. WHEN 点评和奖励完成 THEN 系统应通知任务承接者

### 需求 19：赏金自动计算

**用户故事：** 作为系统，我希望能够根据任务属性自动计算赏金，以确保赏金分配的公平性和一致性。

#### 验收标准

1. WHEN 任务被创建 THEN 系统应根据赏金算法自动计算赏金金额
2. WHEN 赏金算法计算 THEN 系统应考虑基础额度、紧迫性、重要性和预估时长及其对应权重
3. WHEN 任务属性更新 THEN 系统应重新计算赏金金额
4. WHEN 超级管理员修改赏金算法 THEN 系统应只对新创建或未结算的任务应用新算法
5. WHEN 任务已结算 THEN 系统应保持原有赏金金额不变

### 需求 20：赏金算法管理

**用户故事：** 作为超级管理员，我希望能够管理赏金计算算法，以便根据业务需求调整激励机制。

#### 验收标准

1. WHEN 超级管理员访问赏金算法设置 THEN 系统应显示当前算法公式和各参数权重
2. WHEN 管理员修改算法参数 THEN 系统应验证参数的有效性
3. WHEN 管理员保存新算法 THEN 系统应记录算法变更历史和生效时间
4. WHEN 新算法生效 THEN 系统应只对生效时间之后创建的任务应用新算法
5. WHEN 查看历史任务 THEN 系统应显示该任务使用的算法版本

### 需求 21：管理员奖金预算管理

**用户故事：** 作为系统，我希望自动管理管理员的额外奖励预算，以便控制奖励支出。

#### 验收标准

1. WHEN 每月开始 THEN 系统应自动更新所有管理员的额外奖励预算
2. WHEN 管理员发放额外奖金 THEN 系统应从其预算中扣除相应金额
3. WHEN 管理员预算不足 THEN 系统应拒绝额外奖金发放并提示余额不足
4. WHEN 管理员查看预算 THEN 系统应显示当前可用预算和本月已使用金额
5. WHEN 月末结算 THEN 系统应生成管理员奖励支出报告

### 需求 22：通知系统

**用户故事：** 作为用户，我希望及时收到与我相关的任务通知，以便快速响应任务变化。

#### 验收标准

1. WHEN 任务被分配给用户 THEN 系统应发送任务分配通知
2. WHEN 任务接近截止日期 THEN 系统应发送截止提醒通知
3. WHEN 任务依赖被解除 THEN 系统应通知相关用户任务现在可以承接
4. WHEN 任务状态变更 THEN 系统应通知任务发布者和承接者
5. WHEN 管理员发送广播通知 THEN 系统应将通知发送给所有用户
6. WHEN 用户收到通知 THEN 系统应在通知板块显示并支持标记为已读
7. WHEN 通知为广播类型 THEN 系统应在通知中标识发送者为管理员

### 需求 23：任务可视化 - 甘特图

**用户故事：** 作为用户，我希望以甘特图方式查看任务，以便了解任务的时间安排和进度。

#### 验收标准

1. WHEN 用户选择甘特图视图 THEN 系统应以时间轴形式显示所有任务
2. WHEN 任务显示在甘特图中 THEN 系统应根据计划开始和结束时间绘制任务条
3. WHEN 任务存在依赖关系 THEN 系统应用连线显示任务之间的依赖
4. WHEN 任务进行中 THEN 系统应在任务条上显示进度百分比
5. WHEN 用户拖动任务条 THEN 系统应允许调整任务的计划时间

### 需求 24：任务可视化 - 看板

**用户故事：** 作为用户，我希望以看板方式查看任务，以便按状态组织和管理任务。

#### 验收标准

1. WHEN 用户选择看板视图 THEN 系统应按任务状态分列显示任务
2. WHEN 任务卡片显示 THEN 系统应包含任务名称、赏金、截止日期和优先级
3. WHEN 用户拖动任务卡片 THEN 系统应允许在不同状态列之间移动任务
4. WHEN 任务移动到新状态列 THEN 系统应更新任务状态并记录变更
5. WHEN 看板加载 THEN 系统应显示每列的任务数量统计

### 需求 25：任务可视化 - 日历

**用户故事：** 作为用户，我希望以日历方式查看任务，以便按日期安排工作计划。

#### 验收标准

1. WHEN 用户选择日历视图 THEN 系统应以月历形式显示任务
2. WHEN 任务显示在日历中 THEN 系统应在对应日期显示任务标记
3. WHEN 用户点击日期 THEN 系统应显示该日期的所有任务列表
4. WHEN 用户点击任务 THEN 系统应显示任务详情弹窗
5. WHEN 用户切换月份 THEN 系统应加载并显示对应月份的任务

### 需求 26：任务可视化 - 列表

**用户故事：** 作为用户，我希望以列表方式查看任务，以便快速浏览和筛选任务信息。

#### 验收标准

1. WHEN 用户选择列表视图 THEN 系统应以表格形式显示所有任务
2. WHEN 任务列表显示 THEN 系统应包含任务名称、状态、赏金、截止日期、优先级和进度列
3. WHEN 用户点击列标题 THEN 系统应按该列排序任务
4. WHEN 用户使用筛选器 THEN 系统应只显示符合筛选条件的任务
5. WHEN 任务有进度信息 THEN 系统应在进度列显示进度条

### 需求 27：任务进度跟踪

**用户故事：** 作为用户，我希望能够跟踪任务的完成进度，以便了解工作进展。

#### 验收标准

1. WHEN 用户更新任务进度 THEN 系统应允许输入完成百分比
2. WHEN 任务为母任务 THEN 系统应根据子任务进度自动计算总体进度
3. WHEN 任务进度更新 THEN 系统应在所有可视化视图中反映新进度
4. WHEN 任务进度达到100% THEN 系统应提示用户将任务标记为完成
5. WHEN 任务完成 THEN 系统应锁定进度不再允许修改

### 需求 28：自动任务调度

**用户故事：** 作为系统，我希望能够自动调度任务，以便优化任务分配和执行顺序。

#### 验收标准

1. WHEN 任务依赖解除 THEN 系统应自动将任务标记为可承接
2. WHEN 任务截止日期临近 THEN 系统应提高任务的优先级权重
3. WHEN 用户承接任务 THEN 系统应检查用户的工作负载并提供建议
4. WHEN 任务长时间未被承接 THEN 系统应向符合条件的用户推送任务推荐
5. WHEN 任务被放弃 THEN 系统应重新评估任务优先级并通知潜在承接者
