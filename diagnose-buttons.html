🔧 按钮问题诊断工具

第一步：清除缓存

运行以下命令清除前端缓存：

clear-cache-5173.bat

或者手动执行：

cd packages/frontend rmdir /s /q node_modules\.vite rmdir /s /q dist npm run dev
第二步：强制刷新浏览器

在浏览器中按 Ctrl + Shift + R (Windows) 或 Cmd + Shift + R (Mac) 强制刷新页面

🔍 常见问题排查

问题1：userId 和 publisherId 不匹配

症状： isPublisher 显示为 false

原因： 用户ID类型不一致（字符串 vs 数字）

解决： 检查 JWT token 中的 userId 字段类型

// 在控制台运行： const token = localStorage.getItem('token'); const payload = JSON.parse(atob(token.split('.')[1])); console.log('Token payload:', payload); console.log('userId type:', typeof payload.userId);
问题2：Props 在某一层丢失

症状： 某个层级的日志显示 props 为 false 或 undefined

原因： Props 传递链中断

解决： 检查该层级的代码，确保正确传递 props

问题3：浏览器缓存旧代码

症状： 修改代码后没有效果

原因： Vite 缓存或浏览器缓存

解决： 运行 clear-cache-5173.bat 并强制刷新浏览器

问题4：任务状态不符合显示条件

症状： 某些按钮不显示

原因： 任务状态不满足按钮显示条件

按钮显示条件：

  • 发布按钮：status === 'NOT_STARTED' && isPublisher
  • 编辑按钮：isPublisher && isPublishedTasksPage
  • 指派按钮：!assigneeId && (status === 'NOT_STARTED' || 'AVAILABLE')
  • 删除按钮：isPublisher && (status === 'NOT_STARTED' || 'AVAILABLE')

🎯 快速诊断脚本

在浏览器控制台运行以下脚本：

// 检查 token const token = localStorage.getItem('token'); if (token) { const payload = JSON.parse(atob(token.split('.')[1])); console.log('✅ Token payload:', payload); console.log(' userId:', payload.userId, '(type:', typeof payload.userId, ')'); } else { console.log('❌ No token found'); } // 检查当前页面 console.log('📍 Current URL:', window.location.href); // 等待页面加载后检查 props setTimeout(() => { console.log('🔍 请查看上面的日志，搜索：'); console.log(' - [PublishedTasksPage]'); console.log(' - [TaskListContainer]'); console.log(' - [TaskListTable]'); }, 2000);
